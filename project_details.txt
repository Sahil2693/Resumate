================================================================================
PROJECT DETAILS: RESUMATE - AI-POWERED RESUME BUILDER
================================================================================

1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
ResuMate is a modern, full-stack web application designed to simplify the resume-building
process for job seekers. It uses Google Gemini AI to help users generate professional
summaries and work experience descriptions, while providing a real-time, high-quality
visual preview.

2. TECHNOLOGY STACK
--------------------------------------------------------------------------------
Frontend:
- Framework: React.js (Vite)
- State Management: Redux Toolkit
- Styling: Tailwind CSS
- Animations: Framer Motion
- UI Components: Radix UI, Lucide React icons
- AI Integration: @google/generative-ai (Gemini SDK)

Backend:
- Runtime: Node.js
- Framework: Express.js
- Database: MongoDB (Mongoose ODM)
- Authentication: JWT (JSON Web Token) with bcrypt for hashing
- API Utilities: Standardized ApiResponse and ApiError handling

3. KEY FEATURES
--------------------------------------------------------------------------------
- AI Content Generation: Leverages Google Gemini for summaries and descriptions.
- Dynamic Editor: A multi-step form (Personal, Summary, Experience, Projects, Education, Skills).
- Live Preview: Real-time visual feedback of the resume as data is entered.
- Theme Customization: Users can select custom theme colors for their resumes.
- Dashboard: Manage multiple resumes (Create, Edit, View, Delete).
- ATS Optimization: Semantic HTML structure and optimized print styles for better parser compatibility.
- Modern UI/UX: Animated transitions, sticky blurred headers, and responsive design.

4. USER WORKFLOW (START TO END)
--------------------------------------------------------------------------------
Step 1: Landing Page
- User visits the homepage, which features a modern, animated hero section and feature overview.
- User clicks "Get Started" to enter the application.

Step 2: Authentication
- If not logged in, user is redirected to the Auth Page (Sign In/Sign Up).
- Secure login using JWT; session data is managed via Redux.

Step 3: Dashboard
- Upon login, user enters the Dashboard.
- User can see existing resumes or click "Create New Resume".
- Creating a new resume requires a title and takes the user to the Editor.

Step 4: Editing & AI Assistance
- User fills out sections step-by-step.
- In "Summary" or "Experience", the user can click "Generate with AI".
- The system sends a prompt to Gemini AI, which returns professionally written suggestions.
- User selects a suggestion, which automatically populates the form.

Step 5: Preview & Customize
- As the user types, the Preview Page (right side on desktop) updates instantly.
- User can change the resume theme color at any time.

Step 6: Finalization & Download
- Once satisfied, the user can view the final resume in a full-screen mode.
- User can download or print the resume as a high-quality PDF.

5. DATABASE SCHEMA & COLLECTIONS
--------------------------------------------------------------------------------
The project uses the `ai_resume_builder` database in MongoDB with two primary
collections (equivalent to tables in SQL).

1. 'users' Collection:
   - fullName (String): The user's full name.
   - email (String, unique): User's primary email for login.
   - password (String): Hashed password for security.
   - timestamps: createdAt and updatedAt.

2. 'resumes' Collection:
   - title (String): The name of the resume project.
   - user (ObjectId, ref: 'User'): Reference to the owner of the resume.
   - themeColor (String): Hex code or CSS class for the selected theme.
   - Personal Details:
     - firstName, lastName, email, jobTitle, phone, address, summary (Strings).
   - experience (Embedded Array of Objects):
     - title, companyName, city, state, startDate, endDate, currentlyWorking, workSummary.
   - projects (Embedded Array of Objects):
     - projectName, techStack, projectSummary.
   - education (Embedded Array of Objects):
     - universityName, degree, major, startDate, endDate, description, grade, gradeType.
   - skills (Embedded Array of Objects):
     - name (String), rating (Number).
   - timestamps: createdAt and updatedAt.

6. PROJECT ARCHITECTURE
--------------------------------------------------------------------------------
The project follows a modular structure:
- Backend: Implements MVC (Models, Views/Controllers, Routes).
- Frontend: Organized by features (user, resume) and pages (home, dashboard, auth).
- Middleware: Custom auth middleware for protecting private routes.

7. MONGODB CONFIGURATION & DATABASE CONNECTION
--------------------------------------------------------------------------------
To connect and configure the database for a new development environment, follow
these steps:

Step 1: Create a MongoDB Atlas Cluster
- Go to https://www.mongodb.com/cloud/atlas and create a free account.
- Build a new cluster (Shared Tier is sufficient).
- In "Network Access", add your current IP address (or 0.0.0.0/0 for testing).
- In "Database Access", create a new user with "Read and write to any database"
  privileges. Note down the username and password.

Step 2: Obtain Connection String
- Click "Connect" on your cluster, select "Connect your application".
- Copy the SRV connection string (e.g., mongodb+srv://<db_username>:<db_password>@cluster0.example.mongodb.net/?retryWrites=true&w=majority).

Step 3: Configure Environment Variables
- In the `Backend` directory, create or edit the `.env` file.
- Add your connection string to the `MONGODB_URI` variable:
  MONGODB_URI="your_mongodb_srv_connection_string"
- Ensure you replace `<db_username>` and `<db_password>` with your actual
  database user credentials.

Step 4: Database Name
- The project is configured to use a database named `ai_resume_builder`.
- This is defined in `Backend/src/db/index.js` using the `dbName` option in
  `mongoose.connect()`.

Step 5: Verify Connection
- Run the backend using `npm run dev` or `node src/index.js`.
- If successful, you will see: "✅ MongoDB Connected: <cluster_host>"
- If it fails, common causes are:
  - Incorrect IP whitelisting in MongoDB Atlas.
  - Special characters in the password not being URL-encoded.
  - Network/DNS restrictions (try a different network or hotspot).
